[Git] Github 같은 저장소 함께 쓰기(feat.협업하기)

by iOS Develover Fomagran 2021. 5. 13.
728x90
반응형
SMALL

 

 

안녕하세요🍍 Foma🍍 입니다!

 

오늘은 Git으로 같은 저장소를 함께 쓰면서 협업하는 법에 대해서 정리해보겠습니다.

 

바로 시작할게요!

 

(Visual Studio 기준으로 설명드리겠습니다.)

1. 함께 쓸 저장소 만들고 초대하기

 

Github으로 들어가서 New버튼을 눌러 새로운 저장소를 만들어주세요.

 

원하는 저장소 이름을 적고 Create Repository를 눌러줍니다.

 

 

만들어준 저장소의 Settings로 들어가서 왼쪽 탭에 있는 Manage access를 눌러줍니다.

 

 

 

Inviete a collaborator 버튼을 눌러 함께 할 사람을 찾아줍니다.

 

 

함께할 사람을 고르고 Add to this repository를 눌러 추가해줍니다.

 

 

이렇게 하면 초대한 사람이 뜨고 요청을 기다리는 상태라고 뜹니다.

 

2. Git 클론하기 (이미 만들어져있는 저장소 함께 쓰기)

 

만약 제가 만든 저장소가 아니라 상대방이 만든 저장소를 함께 사용한다면 어떻게 해야 할까요?

 

상대방이 만든 저장소로 들어가서 초록색 Code 버튼을 누르고 HTTPS탭을 선택한 뒤 url을 복사해줍니다.

 

 

Visual Studio를 켠 뒤 shift + command + p 를 눌러주세요.

 

그 다음 git이라고 치면 Git:Clone이 뜰거에요. 클릭해주세요!

 

 

위에서 복사한 url을 붙여넣고 엔터를 쳐주고 원하는 곳에 저장합니다.

 

이렇게 하면 오른쪽 하단에 클론한 저장소 켤꺼니라고 물어보는데 Open을 눌러서 켜주세요!

 

3. 새로운 Branch 만들기(나만의 공간 만들기)

 

이렇게 클론한 저장소는 기본적으로 공용으로 사용하는 main 브랜치에 있어요.

 

공용으로 사용하는 저장소를 함부로 바꾸면 안되겠죠?

 

그렇기 때문에 나만 코드를 바꿀 수 있는 공간(Branch)을 따로 만들어줍니다.

 

Git Clone 할 때와 마찬가지로 shitf + command + p 를 눌러주세요!

 

그리곤 Create 라고 치면 Git: Create Branch가 보일거에요 눌러줍니다.

 

 

원하는 이름을 적어주시고 엔터를 눌러주면 브랜치가 생성이 됩니다.

 

4. 나의 코드 저장하고 저장소에 올리기 (Commit & Push)

 

나의 브랜치에서 코드를 작성하고 저장(커밋)해줍니다.

 

왼쪽 탭에 가지모양을 눌러주시고 커밋할 내용을 적은 다음 체크버튼을 눌러줍니다.

 

 

그리고 이렇게 저장한 내용을 푸쉬를 해야 원격 저장소에 저장이 됩니다.

 

 

체크버튼 옆옆에 3개 점모양 버튼을 눌러주시고 Push를 눌러주면 원격 저장소에 내 브랜치의 커밋이 저장됩니다.

 

Github으로 이동해서 실제로 커밋이 되었는지 확인하려면 해당 레포지토리의 Insights 탭을 누르고 왼쪽에 Network로 들어가면

 

새로운 브랜치에서 생긴 커밋들이 파란색 줄로 저장된 걸 볼 수 있습니다.

 

 

5. 공용 저장소에 내 코드를 넣어달라 요청하기(Pull Request)

 

이제 이렇게 작업한 내 브랜치의 코드들을 공용 저장소로 합치려면 어떻게 해야할까요?

 

우선은 같이 작업하는 협업자에게 나의 코드들을 합쳐달라고 요청해야 합니다.

 

요청하는 방법은 해당 레포지토리로 이동해서 Pull requests 탭을 눌러줍니다.

 

아래와 같이 수정한 내용이 있으면 Compare & pull request를 하라고 알림이 떠있을거에요.

 

 

버튼을 눌러서 들어가면 pull request를 만들 수 있는 화면이 나올거에요.

 

원하는 내용을 적고 Create pull request 버튼을 눌러줍니다.

 

6. 공용 저장소와 브랜치 합치기(Git Merge)

 

제가 올린 풀리퀘스트를 협업자가 이상없다고 확인하고 요청에 응답하려면 아래와 같이 Merge pull request 라는 버튼이 생깁니다.

 

(지금은 권한이 저한테도 있어서 그렇지만 원래는 뜨지 않아요.)

 

 

버튼을 눌러주게 되면 아래와 같이 성공적으로 merge가 되었다고 뜹니다.

 

 

아까와 같이 insight에 network로 가서 확인해보면 내 브랜치가 공용 브랜치와 합쳐져있을거에요!

 

7. 합쳐진 공용 저장소 불러오기 (Git Pull)

 

협업자의 코드와 합쳐져 바뀐 공동 저장소의 코드를 확인하고 싶다면

 

위쪽에 있는 Source Control 옆에 세 점 버튼을 누르고 Source Control Repositorie를 눌러줍니다.

 

 

그리곤 아래에서 깃모양 버튼을 눌러줍니다.

 

 

이렇게 하면 다음과 같이 Branch들이 보일거에요. 이 중 공동저장소의 브랜치인 main을 눌러줍니다.

 

 

그리곤 커밋버튼 옆옆 세점 버튼 모양을 누른 뒤 Pull 버튼을 눌러줍니다.

 

이렇게 하면 공동 저장소의 코드로 프로젝트가 바뀔 거에요!

 

8. 내 브랜치에 공동 저장소 코드 불러오기(Git Pull from)

 

협업하는 사람의 풀리퀘스트를 받고 합쳤다면 공동 저장소와 자신의 저장소가 다르겠죠?

 

충돌이 일어나는걸 막기 위해 우리는 공동 저장소의 코드를 나의 브랜치로 가져와야 할거에요.

 

이 때는 똑같이 세점 버튼을 눌러주고 아래로 내려 Pull,Push에 있는 Pull from 버튼을 눌러줍니다.

 

이렇게 하면 공동 저장소의 url이 뜰거에요.

 

 

눌러주면 현재 있는 브랜치들이 있는데 우리는 공동 저장소의 코드가 필요하니 origin/main을 눌러줍니다.

 

이렇게 하면 자신의 브랜치가 공동 저장소의 코드로 바뀌는 걸 볼 수 있습니다.

 

 

이렇게 오늘은 하나의 저장소를 가지고 협업자와 함께 코드를 작성하는 방법에 대해서 알아보았습니다.

 

혹시라도 궁금하신 점이 있거나 틀린 부분이 있다면 댓글로 알려주세요!
